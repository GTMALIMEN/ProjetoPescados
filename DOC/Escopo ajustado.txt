2. Histórico desde 2000: ajuste obrigatório

Esse ponto é essencial.

Para trabalhar com previsão, probabilidade, sazonalidade e machine learning,Economia aplicada, o app precisa guardar histórico. A regra será:

Sempre tentar coletar dados desde 01/01/2000.
Quando a fonte tiver histórico anterior, guardar também.
Quando a fonte não tiver desde 2000, salvar desde a primeira data disponível.

Isso é importante porque algumas fontes têm histórico maior que 2000. Por exemplo, o Comex Stat informa dados mensais de exportação/importação desde 1997 até o ano atual; o FRED tem a série global de fish meal desde janeiro de 1992; a Conab informa pesquisa de preços agropecuários há mais de 30 anos; já no Banco Central e IBGE, o período disponível varia conforme a série/tabela consultada.

Então o correto é:
data_inicio_padrao = 2000-01-01
data_inicio_real = maior histórico disponível por fonte

7. Ponto importante: dado real x dado estimado

Essa regra precisa aparecer no app.

| Tipo        | Exemplo                                       | Como tratar                     |
| ----------- | --------------------------------------------- | ------------------------------- |
| Real        | vendas internas, dólar, importação, população | mostrar como dado medido        |
| Estimado    | consumo provável de proteína por região       | mostrar como estimativa         |
| Previsto    | venda futura, risco de queda, oportunidade    | mostrar como probabilidade      |
| Recomendado | colocar vendedor/promotor/campanha            | mostrar com confiança do modelo |


Exemplo de texto dentro do app:

Esta análise é probabilística. O modelo estima tendência com base em histórico, indicadores econômicos, comportamento regional, preços relativos e dados comerciais. Não representa certeza absoluta.

Isso é obrigatório para o app ficar profissional.


8. Fontes históricas: como usar

Banco Central

Usar para séries macroeconômicas:

dólar;
Selic;
inflação;
juros;
câmbio.

O SGS do Banco Central permite consultar séries econômico-financeiras e automatizar consultas por webservices.

IBGE/SIDRA

Usar para:

população;
renda;
IPCA;
IPCA alimentos;
POF;
Censo;
produção pecuária/aquícola.

O SIDRA reúne séries históricas e pesquisas como inflação, trabalho, rendimento, agropecuária, PIB, Censo, PNAD e POF; a API de agregados permite consultar dados por variáveis, localidades e classificações.

CONAB

Usar para:

preços agropecuários;
milho;
soja;
grãos;
séries históricas de preços.

A Conab informa séries históricas de preços agropecuários e pesquisa sistemática de mais de 112 produtos, com registros por unidades da federação.

CEPEA

Usar para:

boi;
frango;
suíno;
ovos;
milho;
soja;
tilápia;
trigo.

O CEPEA lista indicadores de preços agropecuários para várias cadeias, incluindo boi, frango, milho, soja, suíno, ovos, tilápia e trigo.

Comex Stat

Usar para:

salmão;
bacalhau;
camarão;
merluza;
pescados congelados;
origem dos produtos;
valor FOB;
kg líquido;
preço médio importado.

O Comex Stat é o sistema oficial para extração das estatísticas de comércio exterior brasileiro e disponibiliza dados mensais detalhados desde 1997.

FRED/FMI

Usar para:

fish meal;
commodities internacionais;
preços globais.

A série global de fish meal do FRED/FMI tem dados mensais desde janeiro de 1992 e unidade em dólares por tonelada métrica.
