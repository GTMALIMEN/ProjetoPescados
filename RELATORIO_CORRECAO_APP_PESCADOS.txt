3.4 Simulador IDC

Arquivo ajustado:

- src/services/expansao_service.py

Correção principal:

- adicionada definição canônica final de simular_idc_expansao(), sobrescrevendo versões antigas carregadas por hotfix.

Fórmula oficial aplicada:

IDC = PIB 25%
    + População 30-44 40%
    + População masculina 10%
    + População feminina 0%
    + Restaurantes 10%
    + População 15-29 10%
    + Total PDV 5%

Regras mantidas:

- soma dos pesos precisa fechar exatamente 100%;
- se a soma não fechar 100%, a simulação fica bloqueada e preserva o IDC base;
- fatores são mantidos em escala 0-100;
- receita continua complementar e não altera o IDC base;
- parâmetros antigos ficam aceitos por compatibilidade, mas renda/PIB per capita não entram na fórmula oficial atual.

5. O QUE AINDA NÃO FOI VALIDADO 100%

5.1 Banco real

Não foi possível validar linha por linha os dados oficiais já gravados no PostgreSQL/Supabase porque o ZIP não trouxe dump do banco nem conexão ativa local.

Para validar no seu ambiente, rode:

python scripts/validar_dados_completo.py
pytest -q

5.2 Testes que dependem de banco

No ambiente de correção, o pytest completo ainda depende de:

- driver psycopg instalado;
- PostgreSQL/Supabase configurado;
- tabelas/views criadas;
- dados carregados.

O requirements.txt já possui psycopg[binary]. No seu ambiente, rode primeiro:

pip install -r requirements.txt

Depois rode:

pytest -q

5.3 Testes antigos conflitantes

Existe teste antigo esperando fator_faixa_etaria, mas outro teste mais novo exige que fator_faixa_etaria não exista mais. Como os dois são contraditórios, foi mantida a regra correta atual:

- não usar fator_faixa_etaria;
- usar pct_30_44 e pct_15_29 conforme documentação atual.
